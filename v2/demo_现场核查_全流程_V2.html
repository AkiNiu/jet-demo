P2 · 核查执行 + 判定

请试试以下完整流程

  • 1. 拍照：点"电源核查"→ 选拍照项 → 按快门连续拍（每项最多5张）→ 当前项拍满自动跳下一个
  • 2. 查看/删除照片：点击已拍的项 → 展开照片网格 → 每张显示序号+拍摄者 → 点×可删除指定那张
  • 3. 判定合格：全部至少1张后 → 底部原地展开判定表单 → 选"合格"→ 提交
  • 4. 判定不合格：选"不合格"→ 弹出Bottom Sheet勾选原因 → 点"去拍摄留证照片"→ 取景窗变红重新激活快门 → 拍完点"完成留证拍照"→ 回到判定面板提交
  • 5. 帮别人拍：点"送电痕迹"（李四负责）→ 你不是负责人，拍完后显示"等待李四判定"
  • 6. 直接判定型：点"临电电源"→ 弹出Bottom Sheet直接判定
  • 7. 切换视角：切到"李四视角"看协同效果

两种任务类型

  • 📷 拍照型（电源核查/建设情况/居民入住/充电设施/居民设施）：点击进入拍照模式 → 取景窗+快门连续拍 → 拍完后底部原地展开判定（不跳页）
  • 📋 直接判定型（送电痕迹/临电电源/安全风险）：点击弹出Bottom Sheet判定面板 → 默认不需要拍照，不合格时可补拍

判定交互设计

  • 同一页面完成：拍完后快门区域原地替换为判定表单，取景窗和拍照列表始终可见
  • Radio单选：合格 / 不合格
  • 合格路径极简：选合格 → 提交，不弹窗
  • 不合格用Bottom Sheet：原因勾选（checkbox多选）在覆盖层中，不压缩页面
  • 留证拍照复用取景窗：不合格勾完原因后回到取景窗，快门重新激活，标签变红"不合格留证拍照"，拍完后回到判定面板
  • 底部只有两个按钮：取消 + 提交判定，保持清爽

多人协作提示（已实现）

  • 场景1 — 拍照模式内：slot列表顶部显示协作提示栏（"李四帮拍了：1#开关站外景(2张)…"），每个slot行右侧显示协助者标签（"李四·2"）
  • 场景2 — 主列表：我负责的任务卡片底部展开协助明细行（按人按slot列出）
  • 实时在线状态（PRD需求，Demo未模拟）：多人同时打开同一核查项时，导航栏显示在线头像。实际APP需WebSocket/长连接支持，降级方案为轮询

照片删除权限

  • 核查项负责人（如张三负责电源核查）：可删所有照片（自己的+协助者的），因为负责人要把关照片质量
  • 协助者（如李四帮拍电源核查）：只能删自己拍的照片，不能动负责人或其他协助者的
  • 工作负责人：不操作P2，只在P3做确认/退回。觉得照片有问题→退回整项
  • 删除别人照片时，确认弹窗需注明"这张由XX拍摄"

照片上限策略（双层限制）

  • 客户端软上限：5张/slot — UI引导，拍到5张快门置灰，自动跳下一项
  • 服务端硬上限：10张/slot — 绝对上限，超过直接拒绝上传
  • 5~10张区间：仅在网络延迟导致的多人并发冲突时出现，接受但标记"溢出"，负责人在判定前清理
  • 原则：照片可以多不可以少——多了只需后续清理，少了意味着伪造或返回现场补拍

离线策略

  • 有网：多人协同，实时同步，双层限制生效
  • 断网：回退单人模式，本地拍照存本地，客户端5张上限生效，不做多人协同
  • 恢复：本地队列批量上传，服务端按双层限制逐张校验，超出标记"溢出"

帮拍时负责人已提交判定

  • 协助者正在帮拍 → 负责人已提交判定 → 协助者页面通知条"XX已提交判定，拍摄已结束"
  • 快门禁用，取景窗变灰，仅保留"返回"按钮
  • 已拍未上传的照片仍然上传（照片可多不可少），不丢弃

判定提交与确认流转

  • 试验人员提交判定 → 状态变为"待确认" → P2中不可修改
  • 工作负责人在P3确认 → 状态变为"已确认" → 永久锁定
  • 工作负责人在P3退回 → 照片+判定全清 → P2显示"已退回"+原因 → 试验人员点击后重新核查

P3 · 核查确认

角色：工作负责人

请试试以下流程

  • 1. 先去P2完成几项核查并提交判定
  • 2. 回到P3，看到"待确认"的项
  • 3. 点击待确认项 → 查看照片+判定详情
  • 4. 点"确认通过"→ 该项锁定
  • 5. 或点"退回"→ 填写原因（选填）→ 该项清空，试验人员需重做
  • 6. 回到P2查看退回效果

工作流闭环

  • 分工（P1）→ 执行判定（P2）→ 确认/退回（P3）→ 重做（P2）
  • 试验人员提交后进入"待确认"，自己无法修改
  • 工作负责人确认后锁定，退回后清空重做

退回规则

  • 退回原因选填，不强制
  • 退回后：照片+判定全部清空，负责人不变
  • 试验人员在P2看到"已退回"标记和退回原因
改派功能（退回时更换负责人）暂不做，留V3.1

状态流转全景

  • 未提交 → 试验人员提交判定 → 待确认（黄色闪烁）
  • 待确认 → 工作负责人确认 → 已确认（绿色/红色，锁定）
  • 待确认 → 工作负责人退回 → 已退回（红色闪烁）→ 照片+判定清空
  • 已退回 → 试验人员点击 → 显示退回原因toast → 回到未提交状态重新核查
