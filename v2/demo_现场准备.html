现场准备 — 到场确认

工作负责人到达现场后，确认到场定位、人员到位情况、项目基础信息。

流程位置：作业前准备 → 现场准备 → 建设情况确认 → 核查分工 → ...

页面核心定位

  • 这个页面的核心语义是"你到场了"，不是"填表"
  • 定位确认是页面存在的首要理由，因此从分组中提出，做成顶部 Hero Card
  • 定位同时承担照片水印数据源职责 — 定位文本将用于全项目所有后续检测照片的水印
  • 其他表单项由低代码配置驱动，具体内容可随时调整

定位 = 到场确认 + 照片水印基准

  • 为什么一次定位全局沿用？站房内部GPS信号极差，在室外锁定一次准确定位是唯一可行方案
  • 默认规则：定位成功后，系统自动拼接"定位地址 + 项目名称"生成水印文本
  • 可编辑：用户可点击"编辑"修改水印文本内容
  • 提交后不可修改：水印文本提交后锁死，保证全项目照片水印一致性
  • 定位失败兜底：GPS失败时提供"重试"+"手动输入地址"，不阻塞业务
  • 重新定位：会覆盖已编辑的水印文本

三层视觉重量设计

  • 第一层（Hero Card）：到场定位 — 独立卡片，虚线边框+主色背景，最醒目
  • 第二层（需动手的项）：拍照（工程车辆）、枚举选择（电压等级）— 控件形态自带视觉重量，不用额外标注
  • 第三层（状态确认项）：合格/不合格 toggle — 最轻量，可一键批量操作

排列顺序（强调定位，突出不同）

  • 组间：定位卡片（Hero）→ 项目准备 → 主业人员 → 配合人员，操作重量从重到轻
  • 组内（项目准备）：独特交互前置（拍照→枚举），相同toggle后置（执法设备→配合事项）
  • 用户做完重操作后，一键全部合格收尾

"一键全部合格"（与作业前准备一致）

  • 位置：底部操作区，提交按钮上方
  • 样式：文字链，视觉比提交按钮轻
  • 作用范围：仅作用于 toggle 项（7个状态确认项）
  • 不影响：拍照、枚举选择、定位 — 这些需要用户手动完成
  • 点击后文字变为"全部已确认合格"，颜色变绿

主路径

  • 打开页面 → 点击"确认到场定位" → 定位成功
  • 拍摄工程车辆照片
  • 选择电压等级
  • 点击"一键全部合格" → 7个toggle项翻绿
  • 点击"提交" → 完成
  • 约4-5步操作

已确认设计决策

  • 安措模式已移除：不再在现场准备页面选择安措模式。3.0采用"逐站房独立解锁"机制，安措完成即解锁该站房检测，无需预选模式。该设计属于项目站房安措环节。
  • 闭环设计保留：工程车辆拍照（证明到场）、配合事项验收（确认对方已完成），与作业前准备形成"发出→验收"闭环。

评审确认项

  • 电压等级：保留，无默认值，必须到现场手动确认
  • 工程车辆照片：必拍（1-5张），打水印留痕
  • 中断不保留：退出页面后定位/照片/选择全丢，需重新操作
  • 提交后不可修改：强制要求，一以贯之
vs 2.4系统：2.4系统所有项逐项提交（11项 × 3步 = 33次操作）。3.0定位+拍照+选择+一键合格+提交，约4-5步完成。安措模式选择从现场准备移除，改为项目站房安措环节的逐站房解锁机制。
设计原则：页面体验是"到场打卡"，不是"填表"。控件形态自己说话，不用标签/红星人为强调特殊项。
