P1 — 创建工单 & 派发

设计概要

  • 单页表单，从2.4系统的20字段精简到9字段（基础信息6 + 项目规模4视觉分组 + 派发3）
  • 创建和派发合并为一步操作。Flowable V3.0只有1个节点（create_dispatch），提交即完成。V3.1部署新版流程定义加审批节点
  • 选队伍后自动带出默认工作负责人+队伍人员（PC后台配置），90%情况直接提交，借人时全量添加
  • 工作负责人与队伍人员互斥：同一人不能同时出现在两个位置，工作负责人不在核查分工人员池内
  • V3.0不做：审批、草稿、退回修改、消息通知、排期预览——全部留V3.1

字段裁剪依据（源码追溯+报告实物验证）

保留的字段进报告？说明
项目名称是（封面/表头）核心标识

交互说明

  • 所有下拉选择使用 Bottom Sheet 从底部弹出（MD3规范，触控区域≥48dp）
  • 选队伍面板带搜索框；工作负责人/队伍人员修改时全量不过滤（可跨队伍选人）
  • 项目规模为纯数字文本输入（与实际MetaNumeric组件一致，无+1/-1步进器）
  • 行政区划为纯文本输入（V3.0不用MetaAddress级联组件，不投入开发工作量）
  • 全部字段填写完毕+队伍人员≥1人后"提交派发"按钮激活
  • 提交前弹出确认弹窗展示工单摘要（V3.0不做系统通知，项目经理线下告知工作负责人）

核查分工设计方向（后续模块）

V3.0去掉"组"概念。工作负责人到场后按人分配8项核查任务，全员可见所有任务，干完的人直接帮别人。多人协作同步基于任务ID+人员ID，不依赖组。

P2 — 工单列表

设计概要

  • 顶部Tab筛选：全部 / 已派发 / 已接收（V3.0不做草稿）
  • V3.0用户可见状态只有两种：已派发 → 已接收
  • 右下角FAB按钮（+）直接进入创建页面
  • 示例数据默认为市检（V3.0一期目标适配市检）

工单卡片信息层级

行内容来源
第一行项目名称 + 状态标签创建时填写
第二行项目类型 + 规模（带标签：站X 房X 合建X 箱变X）创建时填写
第三行检测日期 + 行政区划缩写 + 工作负责人创建+派发

对比2.4系统改进

  • 项目规模从"0-0-1-0"无标签 → "站1 房0 合建1 箱变0"带标签
  • 去掉了：项目地址、检测阶段（初检/复检）
  • V3.0无草稿状态，所有卡片均显示工作负责人
  • 默认项目类型从省检改为市检

P3 — 接收确认（工作负责人视角）

设计概要

  • V3.0最小化接收：查看摘要 → 确认接收，一个按钮完成
  • V3.0不做：拒绝/退回（V3.1）、排期预览（V3.1）、消息通知（V3.1）
  • 确认后状态变为"已接收"，可进入现场核查流程
  • 排期冲突通过线下沟通解决（项目经理创建工单前已与工作负责人确认档期）

信息展示

  • 三个卡片分组：基础信息、项目规模、派发信息（全部只读）
  • 检测日期用红色高亮（提醒时间紧迫性）
  • 派发信息包含队伍人员名单（工作负责人可提前了解出场阵容）

接收后数据流

  • 项目规模 → genStationRoomInfoNew() 自动生成站房记录 → 建设情况确认消费
  • 检测队伍+工作负责人+队伍人员 → 核查分工的人员池（按人分配，无组）
  • 项目名称/检测日期/传票号/行政区划 → 最终进入报告
